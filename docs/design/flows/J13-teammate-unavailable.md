# J13 — User returns after teammate becomes unavailable

**Journey:** 19.13 | **Screens:** Teammate/Group → Open seat → Discover → Group invite

## User goal

Keep momentum without discarding friendship or group history.

## Flow

```
Teammate or group detail
  → Mark member temporarily unavailable OR indefinitely
  → Choose: temporary substitute, permanent addition, or play with fewer
  → Set game, role, time, environment needs
  → Discover candidates (still require mutual connection before group invite)
  → Group states temporary vs permanent before outreach
```

## Language

**Open seat** and **temporary availability** — not "replacement" as disloyalty framing.

## Safety

- New members: mutual connection + group acceptance
- Blocked users excluded
- Private notes never shared with candidates

## Success

Group continues; original relationship and history preserved.
