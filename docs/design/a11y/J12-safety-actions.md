# A11y — J12 Safety Actions

## Block

- Menu item: "Block [display name]"
- Dialog: explains immediate silent effect
- Confirm button: "Block" (destructive styling + text)
- Focus returns to prior context or Connections list after block

## Report

- Form: category `select` with visible label
- Description `textarea` optional unless category requires detail
- Submit: "Submit report"
- Success: `role="status"` "Report received. Case reference [id]."

## Safety Center

- Report status table: proper `th`/`td` headers
- No promise of emergency monitoring in heading hierarchy

## Keyboard

- Safety menu reachable from profile and conversation via consistent "Safety actions" button (not icon-only)

## Cognitive

- Plain language; no legal jargon in primary instructions
